CSRF Attack Simulation Panel

Please log in to MusicWave normally in another tab, then come back here and click the buttons to simulate sending unauthorized cross-site requests.

Scenario 1: Cross-Site Login Exploit

Target: login.php (Method: POST)

Attempts to force authentication or pollute the session by sending arbitrary credentials without passing the session CSRF token.

Scenario 2: Unauthorized Account Creation

Target: register.php (Method: POST)

It forces the victim's browser to submit a registration request for a surreptitious malicious account without the user's knowledge.

Scenario 3: Forced Password Overwrite

Target: reset_password.php (Method: POST)

It exploits the recovery session to force a permanent password change by setting the attacker's credentials without a valid CSRF token.

Scenario 4: Password Recovery Flooding (Mail Bombing)

Target: forgot_password.php (Method: POST)

Attempts to abuse the recovery service from a third-party origin to trigger unauthorized password reset emails (Mail Bombing/DoS) without a valid CSRF token.

Scenario 5: Unauthorized Lyrics Ingestion (Stored XSS Attempt)

Target: upload.php (Method: POST)

Attempts to force the ingestion of a new song text containing a malicious JavaScript payload (Stored XSS) on behalf of the logged-in user, bypassing token verification.

Scenario 6: Unauthorized Asset Download / Drive-By Resource Drain

Target: download.php (Method: POST)

Attempts to force the victim's browser to download media assets (e.g., ID 1) in the background. It tests if the backend drops the request when the vital anti-CSRF token is omitted.
